🏗 1️⃣ Core Master Tables
1. ro_master

Stores Regional Offices.

CREATE TABLE ro_master (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(150) NOT NULL,
    code VARCHAR(50) UNIQUE,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

2. report_master

One record per month per RO.

CREATE TABLE report_master (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    ro_id INT NOT NULL,
    month TINYINT NOT NULL,      -- 1-12
    year SMALLINT NOT NULL,
    status ENUM('draft','submitted','approved','rejected') DEFAULT 'draft',
    version INT DEFAULT 1,
    submitted_by INT NULL,
    submitted_at DATETIME NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    UNIQUE KEY unique_report (ro_id, month, year),

    FOREIGN KEY (ro_id) REFERENCES ro_master(id)
);

Why unique key?

👉 Prevent duplicate monthly reports for same RO.

⚙️ 2️⃣ Configuration Tables (Dynamic Structure)

These are managed from separate config page.

3. report_row_config

Stores all rows including section headers and child rows.

CREATE TABLE report_row_config (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL,
    parent_id INT NULL,
    display_order INT NOT NULL,
    level INT DEFAULT 0,
    is_section BOOLEAN DEFAULT FALSE,
    is_editable BOOLEAN DEFAULT TRUE,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (parent_id) REFERENCES report_row_config(id)
);
Important columns:
Column	Purpose
parent_id	For hierarchy
level	For UI indentation
is_section	Full-width header row
is_editable	Prevent editing
display_order	Ordering
4. report_column_config

Dynamic columns like Karnataka, Kerala, etc.

CREATE TABLE report_column_config (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(150) NOT NULL,
    display_order INT NOT NULL,
    data_type ENUM('number','text') DEFAULT 'number',
    is_total_column BOOLEAN DEFAULT FALSE,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
📊 3️⃣ Cell Data Table (Most Important)
5. report_cell_data

This stores only actual editable cells.

CREATE TABLE report_cell_data (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    report_id BIGINT NOT NULL,
    row_id INT NOT NULL,
    column_id INT NOT NULL,
    value DECIMAL(18,2) NULL,
    text_value TEXT NULL,
    updated_by INT NULL,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    UNIQUE KEY unique_cell (report_id, row_id, column_id),

    FOREIGN KEY (report_id) REFERENCES report_master(id),
    FOREIGN KEY (row_id) REFERENCES report_row_config(id),
    FOREIGN KEY (column_id) REFERENCES report_column_config(id)
);
Why separate value + text_value?

Because:

Most columns numeric

Remarks column text

Keeps structure clean

Easier indexing

📝 4️⃣ Submission Remarks Table

You said remarks mandatory at submit.

We should not store that inside report_master directly if we want history.

6. report_submission_log
CREATE TABLE report_submission_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    report_id BIGINT NOT NULL,
    action ENUM('submitted','approved','rejected') NOT NULL,
    remarks TEXT NOT NULL,
    action_by INT NOT NULL,
    action_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (report_id) REFERENCES report_master(id)
);

This gives:

Full audit trail

Approval history

Rejection remarks

Who did what

Enterprise-grade.

SELECT 
    r.id AS row_id,
    r.name AS row_name,
    r.parent_id,
    c.id AS column_id,
    c.name AS column_name,
    COALESCE(v.value, 0) AS value,
    v.remarks
FROM report_master rm
JOIN report_rows r 
    ON r.report_id = rm.id
JOIN report_columns c 
    ON c.report_id = rm.id
LEFT JOIN report_cell_values v 
    ON v.row_id = r.id 
    AND v.column_id = c.id
    AND v.report_id = rm.id
WHERE rm.id = 1
ORDER BY r.display_order, c.display_order;
